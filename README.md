# Минимальные и максимальные цены товара для категории

Тестовое задание на формирование отчёта с ценами на товары

I.  **Склонируйте репозиторий:**

    git clone https://github.com/opasgruz/test-product-prices/ && cd test-product-prices

II.  **Опубликуйте проект локально:**

    make install

III.  **Откройте страницу проекта:**
    [http://localhost:8080/reports](http://localhost:8080/reports)

IV.  **Примечания:**
1) Необходимо партиционирование для таблицы price. Тип RANGE, интервал 1 день.
2) Дневные партиции создавать в команде по cron заранее на 7 дней вперёд.
3) Данные ценами товаров по категории за 7 дней записываются в 1-ю временную таблицу. Агрегированные данные с минимальными и максимальными ценами записываются во 2-ю временную таблицу. Данные для отчёта с разделением товара на 2 строки с минимальной и максимальной ценой записываются в 3-ю временную таблицу. Далее производится экспорт временной таблицы в csv. (Extract -> Aggregate -> Lateral Join -> Export)
4) В названии файла отчёта фигурирует manufacturer_id. Решил убрать данный идентификатор из названия файла, так как отчёт строится только по идентификатору категории товара и производителей в рамках отчёта может быть несколько
5) На странице отчётов перечень процессов формирования из report_process с пагинацией и ссылками на скачивание отчёта. Также 2 кнопки - обновить страницу и сформировать отчёт, а также select для выбора категории товара. По-умолчанию в select всегда указан 1-й элемент.
6) Таблица перечня категорий товаров в задании отсутствует, в рамках тестового задания сделал enum с фиксированными категориями товаров.
7) Если есть готовый отчёт по такой же категории товара, отдавать на фронт сообщение, чтобы скачивали уже имеющийся.
8) Если отчёт по категории формируется, отдавать сообщение что формирование отчёта по данной категории в процессе, ожидайте.
9) Формирование отчета сделал за предыдущие 7 дней, данные за текущий день не включаются в отчёт.
